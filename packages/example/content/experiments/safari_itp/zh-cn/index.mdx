# Safari ITP 影响实验

本实验探索 Safari 的智能防跟踪（ITP）功能如何影响 Web 应用程序和用户隐私。我们将创建一个简单的 Web 应用程序来演示各种跟踪机制，并观察 ITP 如何阻止或修改它们。

## 什么是 ITP？

智能防跟踪（ITP）是 Safari 中的一项隐私功能，用于帮助防止跨站点跟踪。它通过以下方式工作：

1. 限制第三方 Cookie
2. 分区第一方 Cookie
3. 阻止某些类型的跟踪脚本
4. 修改 localStorage 和 sessionStorage 的工作方式

## 我们将测试什么

在本实验中，我们将创建一个 Web 应用程序来演示：

1. 第三方 Cookie 行为
2. 第一方 Cookie 分区
3. localStorage 和 sessionStorage 限制
4. 跨站点请求跟踪
5. 用户指纹识别尝试

## 开始使用

要运行此实验：

1. 打开 Safari 并检查 ITP 设置：
   - 转到 Safari > 设置 > 隐私
   - 查找"跨站点跟踪"设置

2. 访问我们的测试应用：
   - [测试应用](/experiments/safari_itp/zh-cn/1)

3. 尝试不同的场景：
   - 启用 ITP 时
   - 禁用 ITP 时
   - 在不同版本的 Safari 中

## 预期结果

我们预计会观察到：

- 第三方 Cookie 被阻止或分区
- 跨站点上下文中的 localStorage 限制
- 跟踪脚本的行为被修改
- 基于 Safari 版本和 ITP 设置的不同结果
